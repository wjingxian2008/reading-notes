## 2. 有意义的命名
1. 名副其实
2. 避免误导
3. 做有意义的区分
4. 使用读的出来的名字

## 3. 函数  
1. 短小，还要更短小，20行封顶最佳；
2. 只做一件事，做好一件事；
   *要判断函数是否不止做了一件事，就看是否能拆出一个函数，该函数不仅只是单纯的重新诠释其实现*
3. 每个函数一个抽象层级
4. switch语句
5. 参数：0>1>2，避免3，避免输出参数
6. 使用异常替代返回错误码，错误处理就是一件事
